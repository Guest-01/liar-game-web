# liar-game-web

> 웹기술로 온라인 멀티플레이어 게임 만들기

## 게임 소개

> ※ 해외에서는 `Word Wolf` 게임으로 알려져 있습니다

라이어 게임은 한 명의 라이어를 찾아내는 추리 게임입니다. 시민들은 같은 제시어를 받고, 라이어는 다른 단어(또는 아무 단어도 받지 못함)를 받습니다. 플레이어들은 서로의 설명과 토론을 통해 라이어를 찾아내야 합니다.

## 기술적 고민들

### Unity 혹은 JS 기반의 게임 엔진(Phaser.js 등)을 사용할 것인가?

라이어 게임은 별도의 조작이나 캐릭터, 액션 등이 없고 단순 커뮤니케이션 기반의 소셜 게임이라 일반적인 웹 기술로 충분히 구현이 가능하다고 생각했습니다. 실시간으로 진행되는 게임 특성 상 REST API가 아닌 추가적인 실시간 통신이 필요하다는 점만 유의하면 충분히 가능해보였습니다.

### 실시간 통신은 무엇으로 할 것인가?

고전적인 `WebSocket`을 사용할 수도 있으나 조금 더 추상화가 잘 되어있는 `SocketIO`를 사용하기로 결정하였습니다. 웹소켓을 쓴다면 직접 구현했어야할 기능들을 이미 잘 갖추어 놓았기 때문에 개발이 더 쉬워지고 운영 상 리스크가 줄어든다고 판단했습니다.

예를 들면,
- 자동 재연결: 네트워크가 잠깐 끊겨도 자동으로 복구됨.
- 폴백 지원: 브라우저나 프록시가 웹소켓을 막아도 HTTP 롱폴링으로 자동 전환.
- 이벤트 기반 프로토콜: 서버–클라이언트 간 커스텀 이벤트 전송이 편함.
- 룸/네임스페이스: 다중 채널 구조를 쉽게 구성할 수 있음.
- 브라우저/서버 호환성 관리 부담 감소: Socket.IO가 통신 레이어의 복잡도를 대신 처리함.

이런 이점들이 있습니다.

### 프론트엔드 + 백엔드 구성은 어떻게 할 것인가?

초기에는 가장 흔한 Next.js를 통한 풀스택으로 진행하려고 했으나, Next.js 특성상 웹소켓/SocketIO 지원이 난해하여 별도 백엔드 서버를 두는 식으로 선회하였음. 그러다가 프론트엔드를 분리하는 것도 불필요하다고 판단하여 고전적인 SSR 방식인 MVC 패턴으로 템플릿 엔진을 곁들여 뷰도 함께 단일 서비스로 실행하기로 함.

순수 HTML만으로는 실시간 통신, 상태 처리가 어렵기 때문에 EJS 템플릿에 Alpine.js를 추가하여 최소한의 반응성을 확보하였음. 스타일링은 TailwindCSS를 활용함.

최종 스택:
- Typescript
- Express.js + EJS Template
- SocketIO
- Alpine.js
- TailwindCSS + Lucide Icons
